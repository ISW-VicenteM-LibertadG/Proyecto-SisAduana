Instrucciones de navegación del prototipo SisAduana
Acceso al sistema
Abrir el enlace del prototipo en Figma.
Seleccionar Present (▶) para iniciar la navegación interactiva.
En la pantalla de inicio de sesión ingresar cualquiera de los perfiles disponibles para acceder a sus respectivas funciones.

Navegación del rol Funcionario Aduanero
El Funcionario Aduanero es el encargado de registrar el ingreso de pasajeros y gestionar los procesos operativos del control fronterizo.
Flujo recomendado:
Iniciar sesión como Funcionario Aduanero.
Acceder al Dashboard Principal.
Seleccionar Registro de Pasajeros.
Completar los datos solicitados del pasajero.
Si el pasajero es menor de edad, el sistema realizará la validación de la autorización notarial correspondiente.
Continuar con el registro hasta obtener el comprobante de ingreso.
Este perfil también contempla la gestión de declaraciones SAG y otras validaciones propias del proceso de control fronterizo.

Navegación del rol Supervisor
El Supervisor posee funciones de consulta y monitoreo de la operación.
Flujo recomendado:
Iniciar sesión como Supervisor.
Acceder al Dashboard del Supervisor.
Revisar la información consolidada disponible en el panel principal.
Navegar por los distintos módulos de consulta y seguimiento operativo mediante el menú lateral.
Regresar al dashboard utilizando la opción correspondiente del menú.

Navegación del rol Administrador
El Administrador gestiona los usuarios del sistema y supervisa la actividad registrada.
Flujo recomendado:
Iniciar sesión como Administrador.
Acceder al Dashboard Administrativo.
Seleccionar Gestión de Usuarios para crear, modificar o eliminar cuentas.
Utilizar la opción Crear Usuario para registrar nuevos funcionarios del sistema.
Acceder al módulo Logs del Sistema para revisar el historial de operaciones registradas.
Regresar al panel principal mediante el menú lateral.

Consideraciones
Cada perfil posee acceso únicamente a las funcionalidades autorizadas para su rol.
La navegación se realiza mediante los botones interactivos definidos en el prototipo.
Para volver al menú principal, utilizar las opciones del panel lateral o el botón de inicio disponible en cada pantalla.
El prototipo simula el comportamiento esperado del sistema, por lo que algunas operaciones muestran pantallas de confirmación sin realizar cambios reales en una base de datos.
